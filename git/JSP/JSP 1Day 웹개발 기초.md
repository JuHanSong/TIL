# 1. 웹 프로그램 구조

* 웹 클라이언트-웹 서버- DB서버
  * 웹 클라이언트 : 크롬, IE, HTML, 자바스크립트, CSS 등...
  * 웹 서버 : tomcat(JAVA 전용) 다른 프로그램인 경우에는 다른 프로그램이 필요함.
  * DB 서버 : mySQL, OracleDB 등.... [웹서버 - DB서버를 연결을 DB연동이라고 한다.]

* 클라이언트가 요청을 해야 웹 서버가 응답을 해준다 (요청 - 응답)
* request 객체 : 클라이언트가 입력한 값을 서버내에서 받아줄 때 사용하는 객체.

# 2. 웹 서버가 하는 일

* 웹 서버는 단순히 인터넷에 연결되어 있으면서 클라이언트 컴퓨터로부터 요청을 기다린다.
  * 클라이언트 요청 : 문서, 이미지, 사운드. 비디오 등..
  * 웹 서버와 클라이언트는 서로 표준 언어(HTTP) 프로토콜을 사용.
    * HTTP Requst : 특정 파일을 요청.
    * HTTP Response : 찾은 파일을 리턴.

# 3. HTML은 무엇인가?

* 웹 페이지를 기술하기 위한 언어.

* Markup 언어 : 텍스트에 태그를 붙여서 텍스트가 문서의 어디에 해당하는지를 기술.

  ```html
  <title> Web Programming </title>
  ```

  * 태그로 둘러싸인 부분이 문서의 어떤 부분에 해당하는지를 나타낸다.
  * 태그 이름은 대소문자를 구별하지 않는다. 
  * 하나의 요소 안에 다른 요소가 포함이 가능하다.
  * 시작 태그와 종료 태그 사이의 연속된 공백은 하나의 공백으로 취급.

## 1) 요소(elemnt) 

* 시작 태그와 종료 태그로 이루어진 문서의 구성 요소.
* 시작 태그 + 콘텐츠 + 종료 태그로 구성.

## 2) 속성(attribute)

* 요소에 대한 추가적인 정보를 제공.
* 속성은 항상 시작 태그에 기술, 이름 = "값"형태로 기술.
